# AI 吞噬价值链：从"信息检索"到"认知代办"的产品替代规律

> 这篇文章来自一场关于 AI 产品演化的对话讨论，尝试通过"AI 吞噬认知劳动"这个框架，解释为什么 AI 产品总是以一种出人意料的方式替代了那些"本不该被替代"的产品。

---

## 一、AI 的"跨界替代"：干掉的不是竞品，而是行为

互联网有一个规律：一个新产品在诞生时，消灭的往往不是它直接的竞争对手，而是一个完全不在同一生态位置的产品。

AI 也是如此。有了 ChatGPT 之后，我几乎不再用搜索引擎——不是百度，也不再是传统意义上的谷歌。我现在打开谷歌，用的是它的 AI 搜索功能：把问题像发信息一样发出去，它帮我检索，再把整合后的答案还给我。

表面上看，ChatGPT 替代了 Google。但更准确的说法是：

> **它替代的不是一个 App，而是用户原来的一整套行为——关键词搜索、浏览十几个网页、自己比较、自己整合。**

这个区别很重要。真正发生变化的不是"工具"，而是原来那段需要人类自己完成的**信息加工过程**。

---

## 二、信息流动结构的重塑

从信息流动的角度来看，这个变化可以画成这样：

**以前的模式：**

```
信息来源
    ↓
搜索引擎（建索引 / 排序 / 摘要）
    ↓
链接列表
    ↓
人类（自己阅读 → 筛选 → 综合 → 形成判断）
```

**现在的模式：**

```
信息来源
    ↓
搜索引擎
    ↓
检索到的文档
    ↓
LLM（理解 → 推理 → 组织 → 表达）
    ↓
人类（验收 → 纠偏 → 追问）
```

有一件事发生了根本性的变化：**以前"综合信息"这一步是人类完成的，现在是 AI 完成的。** 这是一次角色转移——人类从 Information Processor，逐渐变成了 Supervisor。

### 信息损耗的四个层次

在这个过程中，AI 引入的"信息损耗"并不只是一个笼统的问题，它至少包含四种不同的机制：

| 类型 | 含义 | 举例 |
|------|------|------|
| **压缩损耗（Compression Loss）** | 十篇文章压缩成 300 字，大量细节消失 | 类似 JPEG 压缩，不是错误，但细节没了 |
| **解释偏差（Interpretation Bias）** | 同一句话，AI 选择了一种解释，而忽略了其他 | "This may improve performance" → AI 写成 "This improves performance" |
| **融合偏差（Synthesis Bias）** | 多篇来源互相矛盾时，AI 必须做出取舍，少数派观点消失 | A 说有效，B 说无效，AI 写成"总体上是有用的" |
| **幻觉（Hallucination）** | AI 直接补充了不存在的信息 | 最广为人知的问题 |

这四种损耗，对应着完全不同的产品设计挑战——这是后续很多 AI 产品要解决的核心问题。

## 三、用户行为的两种类型：收敛 vs 发散

观察自己使用 AI 的行为，会发现一个有意思的规律：任务本质上分为两类。

### 收敛型任务（Convergent Tasks）

这类任务有一个"足够好"的答案：

- 写邮件、翻译、修改简历
- 写某个函数的模板代码
- 生成请假单

在这里，AI 扮演的是**执行者（Executor）**，用户提出需求，AI 生成，用户快速验收，流程结束。

```
提出需求 → AI 生成 → 快速验收 → 结束
```

### 发散型任务（Divergent Tasks）

这类任务的目标不是"得到答案"，而是**不断修正自己的理解**：

- 探索一个新的行业趋势
- 梳理一个复杂产品的逻辑
- 构建自己的认知框架

在这里，AI 扮演的是**思维伙伴（Thinking Partner）** 或**认知放大器（Cognitive Amplifier）**，对话本身就是价值，每一轮追问都会产生新的问题。

```
提出观点 → AI 挑战 → 自己修正 → 产生新问题 → 继续深入
```

这个区分很重要，因为它对应着今天 AI 产品正在分化成的两条路线：**完成任务（Do Things）** 与 **促进思考（Think Better）**。

## 四、分析框架：AI 真正吞掉了什么？

以下是一个分析任意 AI 产品的框架：

> **产品原来包含哪些价值？AI 吞掉了哪部分认知劳动？产品未来会变成什么形态？**

我们将这个框架应用于若干具体案例：

---

### ① 搜索引擎（Google / 百度）

**AI 吞掉：** 检索关键词 + 浏览多个网页 + 自己综合信息

**剩下的价值：** 搜索基础设施、实时数据抓取

**未来形态：** Google 从"搜索入口"变成"信息基础设施（Information Infrastructure）"，AI 成为真正的搜索入口

---

### ② 攻略社区（小红书 / 马蜂窝）

以前：看几十篇攻略 → 自己比较 → 规划路线

**AI 吞掉：** 比较 + 行程规划 + 总结

**剩下的价值：** 真人图片、真实体验、最新动态、"值不值得相信"

**未来形态：** 社区从**知识入口**，退化成 AI 的**证据库（Evidence Base）**——AI 告诉你该看哪些帖子，你再去看真人图片和评论。

---

### ③ Stack Overflow

以前：报错 → Google → Stack Overflow → 复制代码

**AI 吞掉：** 找相同报错 + 理解答案 + 修改代码

**剩下的价值：** 海量的历史案例、边界场景的讨论

**未来形态：** 越来越像一个**历史案例数据库**，而不是开发者提问的第一站

---

### ④ 代码编辑器 / IDE（Cursor、Claude Code）

很多人第一反应是"它替代了 VS Code"。但大多数人还是在 VS Code 里写代码，只是多装了 AI 插件。

**AI 真正吞掉的是：** 去 Stack Overflow 搜报错、去 GitHub 找类似实现、自己写模板代码、来回复制粘贴

**更深一步：** 随着对话式 IDE 的出现，AI 甚至开始替代"细节实现"和"框架设计"——把其中的决策和整体方向留给人类判断。

---

### ⑤ 知识管理工具（Obsidian / Notion）

以前：用户负责分类、建双向链接、总结、定期回顾

**AI 吞掉的：** 自动整理 + 自动建立关联 + 自动生成文档

**未来形态：** 从 Knowledge Base（知识存储库）演变成 Thinking Workspace（思维工作空间）——不是存东西，而是陪你思考

---

### ⑥ 表格工具（Excel）

以前：数据 → 写公式 → 透视表 → 分析

**AI 开始吞掉：** 写公式 + 做图 + 找异常 + 总结规律

**未来形态：** 越来越像一个**数据 Agent**，而不只是一个数据表格工具

---

### ⑦ 演示文稿（PowerPoint）

以前：写内容 → 设计布局 → 调字体 → 做动画

**AI 已经可以：** 自动生成 PPT、自动配图、自动排版

**剩下真正的价值：** "你到底想讲什么"——而不是"怎么做动画"

---

### ⑧ 翻译软件

以前：逐句翻译

**AI 吞掉：** 语言转换之后的表达优化（调整语气、本地化、根据对象改写）

**未来形态：** 翻译从"字词转换工具"变成"跨文化表达协作者"

---

### ⑨ 法律数据库

以前：律师查法条 + 查案例 + 比较 + 撰写意见

**AI 吞掉：** 检索法条和案例的大量时间

**律师的时间开始集中在：** 判断策略 + 与客户沟通 + 风险承担

---

### ⑩ 医学文献数据库

以前：医生 PubMed + Google Scholar + 阅读论文

**AI 可以：** 直接总结几十篇相关论文

**医生的判断力集中在：** 这个证据是否适用于眼前的病人

---

## 五、统一规律：AI 总是先吞认知劳动

回顾这十个案例，有一个惊人的共同点。

以前互联网产品的核心价值结构是这样的：

```
数据（Data）+ 工具（Tool）+ 用户完成认知工作（Cognitive Work）
```

例如：
- **Google：** 网页 + 检索工具 + 用户自己思考
- **Obsidian：** 文件 + 编辑工具 + 用户自己整理
- **小红书：** UGC 内容 + 搜索工具 + 用户自己总结攻略

AI 出现之后，产品的结构开始变成：

```
数据（Data）+ Agent（智能代理）+ 用户监督（Human Supervision）
```

中间那一层，从 **Tool（工具）** 变成了 **Agent（代理）**。

而**被 AI 吞掉的**，几乎无一例外都是这一类行为：

> 浏览（Browse）→ 筛选（Filter）→ 整理（Organize）→ 总结（Summarize）→ 组织（Synthesize）

这些行为有一个统一的名字：**Information Processing（信息处理）**。

**留下来的**，几乎都是：

- 创造新的观点
- 提出目标与方向
- 承担责任
- 价值判断与审美
- 最终决策

这些更接近：**Human Judgment（人类判断）**。

## 六、三代产品的演化路径

从这个规律出发，可以勾勒出互联网产品的三代演化路径：

| 时代 | 产品形态 | 例子 |
|------|----------|------|
| 第一代 | 信息平台（Information Platform）| 门户网站、搜索引擎 |
| 第二代 | 工具平台（Tool Platform）| Google Docs、Notion、Figma |
| 第三代 | 智能体平台（Agent Platform）| Cursor、NotebookLM、AI Research Agent |

具体的演化路径，例如：

- **Google 的演化：** 网页搜索 → AI 摘要（AI Overview）→ AI 研究 Agent
- **Notion 的演化：** 文档 → AI 写文档 → AI 帮你管理整个知识库
- **IDE 的演化：** 写代码 → 代码补全 → 整个项目交给 Agent

## 七、结论：下一代产品的竞争终局

过去二十年的互联网产品，大多在帮助人类**获取信息（Access Information）**。

当下这十年的 AI 产品，大多在帮助人类**处理信息（Process Information）**。

那再往后一步呢？

下一代产品的竞争，或许不再是"谁处理信息最快"，而是**"谁能帮助人类形成更好的判断（Support Better Judgment）"**。

这两件事听起来相近，但其实完全不同：

- **处理信息** = 替代人类的信息加工劳动
- **支撑判断** = 帮助人类做出更高质量的决策

如果这个猜想成立，那么 AI 下一个竞争的赛场，将不再是效率，而是**人类认知质量的提升**。而人类真正无可替代的价值，也将进一步聚焦到：

> 提出更好的问题，做出更负责任的判断，创造真正原创的内容。
